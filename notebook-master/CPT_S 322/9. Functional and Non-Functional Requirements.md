### **Functional Requirements**

*Functional requirements* define what a system is supposed to do.
- They describe the tasks, services, or functions the systems must perform
- They are essential for the system to meet the expectations of the users or stakeholders

### **Non-Functional Requirements**

Specifying the quality and constraints of a feature rather than what it should do.

- Performance
- Security
- Usability
- Reliability
- Scalability


### **Kanban**

Japanese manufacturing system using an instruction card

- Visual tool used to manage and organize tasks, ensuring work in progress items are minimized

It can vary in components based on the project

1. *Backlog*
	Role: holds tasks, features, or ideas that have been identified but are not ready to be worked on
2. *To Do/Ready*
	Role: prioritized to be worked on next
3. *Work in Progress*
	Role: Someone is working on this
4. *Review/Testing*
	Role: A reviewer reviews the code and tests are implemented
5. *Closed*
	Role: Item is finished, reviewed, and closed out
6. *Blocked*
	Role: Stalled due to some issue or dependency that prevents them from moving forward
7. *On Hold*
	Role: Temporarily paused since we need more clarification or this is not necessary for now
