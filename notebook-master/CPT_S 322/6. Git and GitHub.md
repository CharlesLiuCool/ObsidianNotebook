- A *version control system (VCS)* lets you track changes to your source code
- This makes it easy for you to recover old documents if you make a mistake
- It simplifies collaboration with other team members

### **Git and GitHub**

- Free and open source software under GNU
- *Distributed Version Control System (DVCS)*: Easy accessibility of project copies
- Distributed aspect makes it a popular VCS

### **Basic Terms**

- *SSH Protocol:* method to secure remote login
- *Repository:* contains project folders
- *Fork:* copy a repository
- *Pull Request:* to request for review and approval
- *Working Directory:* contains files and subdirectories
- *Commit:* snapshot of the project's current state
- *Branch:* separate line of development
- *Merge:* combines changes from one branch to another
- *Clone:* local copy of the remote Git repository
